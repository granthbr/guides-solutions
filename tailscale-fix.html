Resolving Tailscale Firewall Errors on Linux

An interactive guide to fix `iptables` vs `nftables` conflicts.

1. The Problem: Health Errors

On modern Linux systems, you may see a Tailscale health error related to firewall rules. This occurs because Tailscale is attempting to use older `iptables-legacy` commands, but your system defaults to the newer `nftables` framework. The error typically looks like this: 

# Health check:
#       - Tailscale can't reach the configured DNS servers...
#       - adding [-i tailscale0 -j MARK --set-mark...] in v6/filter/ts-forward: ...
ip6tables vX.X.X (nf_tables): MARK: bad value for option "--set-mark"...

2. Choose Your Solution

Solution 1: System-Wide Fix

Switch your system's default firewall to `iptables-legacy`. This is the most common and straightforward fix.

Recommended

Solution 2: Tailscale-Only Fix

Configure Tailscale to generate `nftables`-compatible rules, leaving your system default unchanged.
